# gc-lang v0.2.0 — The Collector

**The core release.** v0.1.0 was scaffold — structure and quality gates, no domain
logic. v0.2.0 is the garbage collector itself: a safe, handle-based tracing
mark-and-sweep heap for interpreted-language runtimes. Cycles are reclaimed, freed
slots are reused, and a steady-state collection allocates nothing. The public surface
is six items and no runtime dependencies. Zero `unsafe`.

## What is gc-lang?

A tracing garbage collector an interpreted-language runtime can drive directly.
Allocate an object into a `Heap<T>` and hold the returned `Gc<T>` handle; objects store
handles to point at one another; hand the runtime's roots to `collect` and everything
no longer reachable — whole cycles included — is freed. It owns storage and reclamation
only: no value representation, no interpreter, no policy on when to collect.

## What's new in 0.2.0

### `Heap<T>` — the garbage-collected object store

The heap is a slot store with a free list. Allocation reuses a slot freed by an earlier
collection when one is available and grows the backing store otherwise; resolution is a
direct slot lookup.

```rust
use gc_lang::{Gc, Heap, Trace, Tracer};

enum Value {
    Number(f64),
    Pair(Gc<Value>, Gc<Value>),
}

impl Trace for Value {
    fn trace(&self, tracer: &mut Tracer<'_>) {
        if let Value::Pair(a, b) = self {
            tracer.mark(*a);
            tracer.mark(*b);
        }
    }
}

let mut heap = Heap::new();
let one = heap.alloc(Value::Number(1.0));
let two = heap.alloc(Value::Number(2.0));
let pair = heap.alloc(Value::Pair(one, two));
let unused = heap.alloc(Value::Number(3.0));

let stats = heap.collect([pair]);   // pair is the only root
assert_eq!(stats.freed, 1);         // unused was unreachable
assert!(heap.get(one).is_some());
```

The full surface: `new`, `with_capacity`, `alloc`, `try_alloc`, `get`, `get_mut`,
`contains`, `len`, `is_empty`, `capacity`, `collect`, plus `Default` and a shape-only
`Debug`.

### `Gc<T>` — a generation-stamped handle

A `Gc<T>` is eight bytes: a slot index plus a generation stamp. It is `Copy`, and
`Eq` / `Ord` / `Hash` for **every** `T`, so it works as a map key regardless of what it
points at. The generation stamp is the safety mechanism — a slot's generation advances
when it is reused, so a handle to a collected object no longer matches whatever now
lives in that slot. It resolves to `None`, never to an unrelated value. There is no
use-after-free to have.

```rust
# use gc_lang::{Heap, Trace, Tracer};
# struct Node; impl Trace for Node { fn trace(&self, _: &mut Tracer<'_>) {} }
let mut heap = Heap::new();
let h = heap.alloc(Node);
let _ = heap.collect([]);        // reclaims h's object
assert!(heap.get(h).is_none());  // the old handle stays dead, no panic
```

### `Trace` / `Tracer` — the reachability contract

An object type becomes collectable by implementing `Trace`. During the mark phase the
collector calls `trace`, and the object reports each handle it owns by calling
`Tracer::mark`. Marking keeps an object alive; marking a stale handle is harmless (the
collector ignores handles that name no live object); marking extra at worst keeps
something alive one cycle longer. Leaf types implement `trace` as an empty body.

### `collect` — tracing mark-and-sweep

`collect(roots)` marks everything reachable from the roots by walking the handles each
object reports, then sweeps everything it did not mark: the object is dropped, its
slot's generation is advanced, and the slot is returned to the free list. Cycles
terminate because each object is marked at most once. This is the case reference
counting cannot handle:

```rust
use gc_lang::{Gc, Heap, Trace, Tracer};

struct Node { link: Option<Gc<Node>> }
impl Trace for Node {
    fn trace(&self, tracer: &mut Tracer<'_>) {
        if let Some(link) = self.link { tracer.mark(link); }
    }
}

let mut heap = Heap::new();
let a = heap.alloc(Node { link: None });
let b = heap.alloc(Node { link: Some(a) });
heap.get_mut(a).unwrap().link = Some(b);  // a <-> b, unreachable from outside

assert_eq!(heap.collect([]).freed, 2);    // the whole cycle is reclaimed
```

The pass returns `CollectStats { live, freed }`. Cost is `O(reachable)` to mark plus
`O(slots)` to sweep; the mark queue and mark bitset are retained between calls, so a
steady-state collection does not allocate.

### Property-tested reachability

`tests/proptests.rs` checks the collector's core invariant against an independent
breadth-first walk over arbitrary generated graphs — arbitrary node counts, arbitrary
edges (cycles and self-loops included), arbitrary root subsets:

- After a collection, an object is live **iff** the reference walk reached it.
- `len()` equals the survivor count, and `live + freed` equals the original population.
- A second collection with the same roots frees nothing new and disturbs no survivor.

The reference walk is deliberately not the collector's own algorithm, so agreement
between the two is evidence, not a restatement.

### Benchmarks

`benches/bench.rs` (Criterion) covers allocation (fresh slot and free-list reuse),
handle resolution, a single collection over a reachable tree at 1k / 10k / 100k nodes,
and a steady-state allocate/collect loop. Representative local means (release build,
WSL2 Ubuntu on Windows x86_64):

| Operation                    | Cost       |
|------------------------------|------------|
| Handle resolution (`get`)    | ~0.6 ns    |
| Allocation, reused slot      | ~2.3 ns    |
| Allocation, fresh slot       | ~6.7 ns    |
| Collection                   | ~12 ns/obj |
| Steady-state alloc + collect | ~28 ns     |

Collection scales linearly with heap size (≈10.8 µs at 1,000 nodes, ≈1.24 ms at
100,000). Reproduce with `cargo bench`.

### Examples

Three runnable examples in `examples/`:

- `cycle_collection` — reclaims a reference ring that reference counting would leak.
- `mini_interpreter` — a GC'd value heap rooted at an interpreter's operand stack.
- `object_graph` — shared subgraphs surviving as long as any root reaches them.

  the declared `rust-version` (1.85).

## On `arena-lang`

The roadmap anticipated wiring `arena-lang` at this stage. It was evaluated and
deliberately not wired: `arena-lang` is append-only — it never reclaims an individual
slot — while reclaiming and reusing slots is exactly what a collector does. The heap
owns its own slot store with a free list instead. No hard task was deferred; the
dependency was not a fit for the design (recorded in `dev/ROADMAP.md`).
